**Назначение образа:** web-сервер на базе `Apache 2` и `PHP 8.2`

**Родительский образ:** `alt:p10`

**Название образа:** `alt-p10-web`

**Версия/тег образа:** `1.0.0`

**Варианты использования:**

1. Запуск на хосте и доступ с этого же хоста по адресу `http://localhost:8080`.

2. Запуск на хосте с возможностью доступа к контейнеру с других компьютеров вычислительной сети.

3. Прикрепление к контейнеру внешнего каталого с php-скриптами.

**Причины использования `alt:p10`** как базового образ (`FROM alt:p10`) следующие:

1. Прямая сертификация Docker-образа `alt:p10` отсутствует, но его использование на объектах КИИ может быть обосновано через сертификацию средств контейнеризации и хостовой ОС в соответствии с требованиями Приказа ФСТЭК №118.

2. Использование репозиториев alt-linux при установке apache 2 и php 8.2.

**Недостаток:** Большой размер результирующего образа, ~ 1 Gb.

| `./html` | **каталог, копируемый в образ по умолчанию** |
| `./html/default-styles.css` | файл стилей |

| `./html/phpinfo.php` | действующая конфигурация php 8.2 |
| `./scripts/` | **отладочный каталог, для прикрепления во время запуска контейнера** |
| `./scripts/index.php` | исполняемый файл главной страницы |
| `Dockerfile` | сценарий создания образа |
| `README.md` | файл, который Вы сейчас читаете |


### Сборка образа из Dockerfile
```bash
# Сборка образа с тегом
$ docker build -t alt-p10-web:1.0.0 .

# Запуск контейнера, с пробросом портов
# для просмотра через браузер на том же компьютере,
# используя адрес "http://localhost:8080" 
$ docker run -d -p 8080:80 alt-p10-web:1.0.0

# Запуск контейнера с пробросом порта на все интерфейсы хоста
# , чтобы была возможность подключаться к нему с других компьютеров
# в сети
docker run -d -p 0.0.0.0:8080:80 alt-p10-web:1.0.0


# Запуск контейнера с прикреплением внешнего каталога с php-скриптами
docker run -d -p 0.0.0:8080:80 -v ./attach_web:/var/www/html alt-p10-web:1.0.0

# Выяснить id/хеш-контейнера
$ docker ps

# Подключение к командной строке контейнера
$ docker exec -it <container-id> /bin/bash

# Остановить контейнер
$ docker stop <container-id>

# Удалить контейнер
$ docker rmi <container-id> -f
```

### Расширение функциональных возможностей
Выполнять перед сборкой образа

1. Изменение конфигурации web-сервера Apache 2 `./config/httpd2.conf`

2. Изменение конфигурации PHP 8.2 `./config/php.ini`

3. Для установки дополнительных компонент, в секцию 1 добавте,
или иключите из списка:
# [Секция 1]
# Установка необходимых пакетов
    ...
    apt-get install -y \
        apache2-full \
        apache2-mod_php8.2 \
        php8.2 \
        php8.2-gd \
        php8.2-mbstring \
        php8.2-devel \
        php8.2-libs \
        php8.2-pdo \
        php8.2-pdo_pgsql \
        php8.2-pgsql && \
    ...

После необходимых изменений выполните сборку образа как описано выше.
